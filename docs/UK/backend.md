# Реалізація апі

Зміст:

- Опис
- Компоненти
- Сутність Carrier

## Опис

Застосунок складається з одного блоку - підрахунок вартості перевезення в залежності від ваги посилки.  

## Компоненти

- Сутність Carrier
- Служба CarrierService
- Http контролер з двома точками входу:
  1. Відображення html форми
  2. Post Api отримання даних з форми и видачи результату підрахунку
- Репозиторій Carrier

## Сутність Carrier

Сутність Carrier представляє собою перевізника, зі всіма його можливими атрибутами і правилами. Це найголовніша частина блоку. Саме тут знаходяться усі правила і логіка.

Файли розташовані в каталозі [Domain](./../../app/Domain/Carrier/).

Валідація відбувається в середині VO, які представляють атрібути Carrier. Усі слої застосунку мають використовувати VO для передачи інформації від одного шару до іншого. Це гарантує правильність вхідних даних та іх не зміну під час обробки.

Підріхунок відбувається в методі `calculateShippingPriceByWeight`. Для підрахунку використовується поле `$calculator` якє є імплементацією `ShippingCostCalculatorInterface`. Таким чином реалізація підрахунку відокремлюється від класу Carrier. Реалізація калькуляторів знаходиться в шарі [Application](./../../app/Application/CarrierService/ShippingCostCalculators/). Але це не обов'язково. Ці файли можна(і можливо варто) перенести в Domain.

## Служба CarrierService

За роботу з сутність Carrier відповідає служба CarrierService. Вона знає як створити, змінити та знайти потрібного Carrier.

Для роботи з запитами до неї, служба використовує команди. Кожна точка входу (Http, cli та інші) мають використовувати команди для користування сервісом.

Команда - це клас, який приймає дані, перевіряє їх і приводить в такий стан, щоб служба або сутніть могли ними користуватися. Для зменьшення можливих варінтів, дані приводяться до строки. Приклад команди - [CalculateCommand](./../../app/Application/CarrierService/CalculateShippingCosts/CalculateCommand.php). Вона приймає дані або через конструктор або через метод `fromHash`. В першому випадку користувач (контролер) має привести вхідні дані до строки (інакше буде помилка при створенні команди). В другому випадку користувач, не обробляючи вхідні дані завсім, передає їх у вигляді масиву (який є по суті словником).

Команди - є перша ступінь валідації даних. Усі дані приводіться до строки і вже потім потрапляють в VO сутностей де проводиться подальша перевірка.

Усі користувачи служби мають використовувати команди для створення структур даних. Прикладом може бути назва поля [html форми](./../../resources/views/calculate.blade.php), ім'я якої записано в команді як `public const slugField = 'carrier'`.

```blade
<label for="{{ $carrier_slug_field }}">Carrier:</label>
```
